## Catalog Index

Location: `data/catalog-index.json`

Auto-generated by `scripts/build-index.js`. Do not edit manually.

| Field | Type | Description |
|-------|------|-------------|
| `lastBuilt` | string | ISO timestamp of last build |

| `stores` | array | `{ slug, name, flag, count }` per store |
| `categories` | array | `{ slug, name, count, icon }` per category |
| `brands` | array | `{ name, count }` per brand |

## Size Formats

Sizes vary by store. The scraper captures what's on the page:

| Store | Format | Examples |
|-------|--------|----------|
| END. Clothing | EU/UK prefix | `EU 42`, `UK 8`, `S`, `M`, `L` |
| Foot Locker | Bare EU numbers | `40`, `41.5`, `43` |
| Nike | US/EU prefix | `US 10`, `EU 44` |

All sizes are stored as strings. Filtering should normalize before comparing.

## Status Values

| Status | Meaning |
|--------|---------|
| `active` | Product is live and available |
| `sold-out` | Product exists but all sizes sold out |
| `expired` | Sale has ended or link is dead |
| `archived` | Manually removed from display |

Only `active` products are counted in the catalog index and displayed on the site.
